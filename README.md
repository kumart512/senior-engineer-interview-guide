You may see topics like:

- Hash maps
- Two pointers

- Stacks and queues
- Trees and graphs
- Recursion and backtracking

### 🏗️ System Design

This section explains how to think about large software systems.

You may find coverage for:

- API design
- Load balancing
- Caching
- Databases
- Rate limiting
- Search systems
- Messaging
- File storage

Focus on how parts of a system fit together and how traffic moves through them.

### 🧱 Software Architecture

This section helps you build a clear picture of app structure.

Topics may include:

- Service boundaries
- Monoliths and microservices
- Data flow
- Error handling
- Versioning
- Deployment flow
- Monitoring
- Reliability

Use this part to explain how you would keep a system easy to change and maintain.

### 🌐 Distributed Systems

This section covers how multiple machines work together.

You may see:

- Replication
- Consistency
- Partitioning
- Failover
- Message queues
- Event flow
- Leader election
- Timeouts and retries

These ideas matter in interviews for senior roles because they show how you handle scale and failure.

### 💬 Behavioral Questions

This section helps you explain your work in a clear way.

Prepare stories about:

- Leading a project
- Handling conflict
- Fixing a hard bug
- Working across teams
- Making tradeoffs
- Learning from failure
- Mentoring others
- Handling pressure

Use short examples with a clear problem, action, and result.
